## 一、核心概念：镜头切 vs 镜头运动（多镜头判定根基）

这是多镜头标注最基础、最容易出错的概念，基础规则文档未专门展开，此处详述。

### 1.1 镜头切（硬切）— 这才是多镜头

**定义**：画面瞬间跳变到另一角度/场景，通常来自**不同机位**的摄像机，后期剪辑切换。

**判定（满足其一即为镜头切）**：
1. **非一镜到底** → 有镜头切。
2. **硬切检查**：画面「忽然变换」、无过渡动画、瞬间从 A 跳到 B → 即硬切，即镜头切。

**典型场景**：一场戏立 3 台机位（近/中/全），后期来回切 → 每次切换 = 一个镜头切。

### 1.2 镜头运动 — 不是多镜头

**定义**：同一台摄像机连续物理移动/变焦（推、拉、摇、移、跟），画面**连续变化**无跳跃。

> 运镜类型（推拉摇移跟）的具体定义见基础规则文档「运镜分类」，本文档不重复。

### 1.3 混淆的严重后果

- 把镜头运动当镜头切 → 错误把单镜头拆成多 shot，标注全错。
- 「镜头切」与「镜头运动」**绝对禁止混用**。
- 记忆口诀：**连续变化 = 运动 = 单镜头；突然跳变 = 切 = 多镜头。**

## 二、Shot（镜头）格式规范

### 2.1 基本结构

```
视频包含[N]个镜头。

<Shot 1> 从X秒开始，到Y秒结束
景别 + 机位/视角 + 运镜状态 + 镜头
[描述：什么人/东西、做什么、说什么、发出什么声音]

<Shot 2> 从X秒开始，到Y秒结束
镜头切至 + 景别 + 机位/视角 + 运镜状态 + 镜头
[描述...]
```

> 「景别 + 机位/视角 + 运镜状态 + 镜头」的标准格式与示例见基础规则文档 `<Scene>` 标注规范，本文档不重复。

### 2.2 镜头切字规则

- **第 1 个镜头**：不写切字，直接「XX镜头」。
- **第 2 个及之后**：开头必须写切字（「镜头切至XX镜头」「切为XX镜头」）。
- **删除「通过硬切」四字**：原始文本常写「通过硬切切至XXX」，审核删「通过硬切」，只留「镜头切至XXX」。
- 带**特效转场**（叠画/滑动/弹入等）时保留特效描述：「通过叠画转场切至XXX镜头」。

### 2.3 镜头内运镜描述（不新开 shot）

同一 shot 内有运镜（推/拉/摇/移/跟）→ **不新开 shot**，在当前 shot 内按时间顺序描述：

```
镜头切至近景摇镜头
[描述摇的过程中看到什么]
镜头快速推进变为正面全景
[描述推进后的画面]
```

**关键**：同 shot 内运动用「镜头运动成」「变为」「随着镜头…」，**绝对不能用「镜头切」**。

### 2.4 时间戳校验

- 每个 shot 起止时间须与实际切点一致，误差 ≤ **0.5 秒**，超则改。
- **结尾时间**最易出错（AI 文本常比实际短），务必核对。
- 单位注意：`0:04` = 4 秒，不是 0.04 秒。
- **分/秒格式歧义**：`1.6` 表示「1分6秒」，不是「1.6秒」；`6.1` 表示「6秒多（约6.1秒）」。秒数必须与视频实际一致，不能视频是 7 秒你写成 8.5。

## 三、各字段的多镜头差异点

> 各字段的基础标注规范（格式、必写项、禁忌）均见基础规则文档对应标签章节，此处只列多镜头场景下的**差异与补充**。

### 3.1 Header（变速定位）

- 基础规则见基础文档 `<header>`：正常播放留空；变速须按格式标注。
- **多镜头特有**：变速片段用「开头/中间/结尾部分」定位，**不能用「镜头1、镜头2」**（Header 在镜头描述之前，此时尚无镜头概念）。对应 shot（Scene）中也须标明该片段是快速/慢速/延时。
- **抽针/定格（stop-motion）+ 延时**：若视频速度实际正常（只是有定格/抽针画面展示），**不要写「慢速」**；只在对应 shot 描述「画面有抽针/定格展示」即可，Header 不标变速。

### 3.2 Role

- **数量一致**：Header 写「视频中有 X 个主要角色」→ 下面 ID 须是 X 个。改动 ID 数量须同步改 Header。（基础规则亦有，此处强调多镜头下更易漏。）
- **ID 顺序（多镜头特有要求）**：须按 A、B、C **连续**排列，**不能跳字母**（ABD 不行）。
  > ⚠️ 注意：基础规则单镜头场景写明「无需关注顺序要求」；多镜头按培训要求须 ABC 连续。两者适用场景不同，多镜头以本要求为准。
- **核心对象可以是物品**：对讲机、工具等聚焦物品也可为主要角色；只露一只手/手指无其他特征 → 不算。
- **同一人换装**：同一人换衣服 → 用**同一个 ID**，不新增；在 Role 描述注明服装变化时间点（`在N~M秒穿XX，在P~Q秒穿YY`），对应 shot 中也注明。

### 3.3 Background

- **每个 shot 必须带背景**：多镜头下每个 shot 配一个背景编号（`<BG_1>`、`<BG_2>`…），AI 漏了须补，**绝对不能删**。
- **跨镜头背景判断**：
  - 明显差异（方位大变、拍到不同区域）→ 不同背景，分别描述。
  - 无明显变化或背景缺乏有效信息 → 视为同一背景，Scene 中不重复描述，但 Background 字段提一嘴。
- **水印/Logo（多镜头特有处理）**：

  | 水印类型 | 处理方式 |
  |---------|---------|
  | 从头固定不变 | 仅首个镜头背景描述一次（位置+内容），后续不提 |
  | 中途出现/消失/变化 | 在对应 shot 描述（「哪边变在哪边提，没变不提」） |
  | 动态贴片文字（随台词/事件） | 在对应 shot 描述出现/消失，**不写 Background** |

  > 水印的基础描述要求（位置+文字内容）见基础规则文档 `<Background>`。

  **🆕 动态贴片文字标注规则**（07011 补充，源自飞书培训截图）：

  - **位置**：在对应 shot 的 Scene 描述中写明，不写 Background（非静态水印）。
  - **内容要素**：须如实描述贴片的**位置、底色、文字内容**；若伴随音效，用 `<sfx>` 紧跟描述；贴片**消失**也需交代（如「随后贴片文字消失」）。
  - **格式模板**：`[位置]出现[底色]字的"[文字内容]"的贴片，同时伴随<sfx>"[音效内容]"的音效</sfx>，随后贴片文字消失……`
- **平台 logo 认不出**：若无法确定具体平台，直接写「平台名 + 颜色」即可，例：「优酷蓝色 logo」，不必强行辨认。

### 3.4 Style

- 写实/非写实判定、游戏=非写实等基础规则见基础文档 `<Style>`，本文档不重复。
- **风格不一致的视频（多镜头特有）**：前半写实后半非写实 → 描述为「前半段为写实风格，后半段为非写实的XX风格」。**不能用「镜头1、镜头2」定位**（Style 在镜头描述之前），用「前半段/后半段/中间部分」。

### 3.5 Scene（重点）

#### 3.5.1 格式与结构

- 开头写「视频包含[N]个镜头」。
- 每个 shot 包裹：`<Shot N> 从X秒开始，到Y秒结束` → 内容 → `</Shot N>`。
- 开头结尾 shot 编号须一致（Shot 3 结尾须 `</Shot 3>`）。

#### 3.5.2 镜头切 vs 镜头运动（用词）

- 换镜头 → 「镜头切至」「切为」；同镜头内运动 → 「镜头运动成」「变为」。绝禁把运动写成「镜头切」。

#### 3.5.3 跨镜头台词拆分

- 一句话跨两个 shot → 按**每个 shot 内实际听到的台词**拆分，不把整句塞进一个 shot。

#### 3.5.4 分屏画面

| 情况 | 镜头数 | 处理方式 |
|------|--------|---------|
| 景别/角度一致 | 1 个镜头 | 分左右/上下写，一个 shot 两个背景 |
| 景别/角度不一致 | 2 个镜头 | 分别描述左右/上下各是什么镜头 |

#### 3.5.5 特殊案例

- **画面卡顿但内容不变**：角度、背景都没变，仅播放卡顿/跳帧感 → 视为 **1 个镜头**。
- **镜头结尾被遮挡变全黑**：Shot 内要描述「画面逐渐被遮挡/变全黑」的变化过程，不能只写开头不看结尾。
- **PPG**：见第六节废弃条件。

#### 3.5.6 术语 nuance 补充（多镜头易错）

- **过肩镜头不限于肩膀**：从角色头/脖后、或透过某人拍其后方人群等机位，只要是对话/前后景关系的「过肩」构图，均算过肩镜头。

#### 3.5.7 Scene 审核清单（multi-shot 逐项检查）

> 以下为 multi-shot 场景下 Scene 字段的**逐项审核要点**，审核时逐条核对。

| # | 检查项 | 要点 | 对应详细规则 |
|---|--------|------|-------------|
| 1 | 镜头描述准确性 | 是否简明扼要描述了具体什么镜头（景别+机位+运镜） | §2.1 基本结构 / 基础规则 `<Scene>` |
| 2 | 运镜 vs 画面相符 | 相机运镜的描述是否跟画面内容一致（不是照抄源文本，而是以实际视频为准） | §1.3 混淆 + 易错点#13 |
| 3 | 时序性（运镜） | 是否按照时间顺序变化来描述运镜，与人物景交互融合自然 | §2.3 镜头内运镜 |
| 4 | 时序性（交互） | 人物景交互是否按时序准确描述 | 基础规则 `<Scene>` 注意点#1 |
| 5 | 角色指代准确性 | 角色引用 `<ID_x>` 是否正确，尤其 shot 数/人数较多时容易出错 | §3.2 Role + 基础规则 `<Scene>` 注意点#2 |
| 6 | **切字规范** | 从第 2 个 shot 起，首句是否有明确的「镜头切换/镜头切至」字样 | §2.2 切字规则 |
| 7 | 切换点位一致性 | 切镜/场景切换的时间点是否跟视频画面实际一致 | §2.4 时间戳校验 |
| 8 | **跨镜头角色绑定** | ⭐ **跨镜头角色的绑定是否正确**。典型：**正反打镜头**（over-the-shoulder reverse shots）中两个 shot 的角色位置、朝向、ID 引用须严格对应实际画面——shot A 中 `<ID_A>` 在左前景、`<ID_B>` 在右背景 → shot B 切换后应反过来。**这是多镜头高频易错点，须逐一核实** | 🆕 |
| 9 | **跨镜头衔接** | ⭐ 各 shot 之间的描述衔接是否正常、连贯（无跳跃、无矛盾、无重复）。重点核查：上一 shot 结尾状态 → 下一 shot 开头状态是否能逻辑衔接；同一动作是否被错误地拆分或重复描述 | 🆕 |
| 10 | 特殊转场描写 | 泼出、百叶窗等特殊转场手法是否已写出（带特效转场格式：`通过XX转场切至YY`） | 第四章 转场类型 |
| 11 | **跨镜头台词与字幕** | ⭐ **台词横跨多镜头时**：每个 shot 只描述实际听到的部分，不得在未听到的 shot 写完整句；**字幕在多个镜头同时存在时**：每个 shot 都需描述该字幕（字幕是画面元素） | 🆕 |

> 画外音、台词/字幕/声音标签等 Scene 通用规则见基础规则文档 `<Scene>`，本文档不重复。

## 四、转场类型（多镜头特有）

| 转场类型 | 说明 |
|---------|------|
| 硬切 | 瞬间跳变无过渡，写「镜头切至XX镜头」 |
| 叠画 | 两画面短暂重叠后切换 |
| 滑动 | 新画面从一侧滑入覆盖旧画面 |
| 弹入/淡入淡出 | 画面逐渐显现或消失 |
| 特效转场 | 特定视觉特效过渡 |
| 万花筒转场 | 万花筒效果切换 |
| 百叶扇转场 | 百叶窗效果切换 |
| 运动转场 | 甩镜、剧烈旋转、跳切等方式 |

带特效转场格式：`通过XX转场切至YY镜头`（与硬切的「只写镜头切至」区分）。

## 五、易错点汇总（多镜头特有）

| 序号 | 易错点 | 正确做法 |
|------|--------|---------|
| 1 | 把镜头运动当成镜头切 | 连续变化=运动，突然跳变=切 |
| 2 | 漏写镜头切字 | 除首镜头外，每个 shot 开头必须写 |
| 3 | 保留「通过硬切」四字 | 删除，只留「镜头切至」 |
| 4 | 漏删 shot 的背景 | 每个 shot 必须有对应背景 |
| 5 | ID 数量改动后忘同步 Header | 前后一致 |
| 6 | ID 跳字母（ABD） | 必须 ABC 连续 |
| 7 | 同一人换装开新 ID | 同人同 ID，描述中写服装变化 |
| 8 | 水印变/没变都反复写 | 不变仅首镜头一次，变了的在对应镜头写 |
| 9 | 场景音全堆最后一句 | 分配到对应各背景 |
| 10 | Style/Header 用「镜头1、镜头2」定位 | 用「前/中/后段」，这些字段在镜头之前 |
| 11 | 一句话跨镜头不分拆 | 按每个 shot 实际听到的拆分 |
| 12 | 同 shot 内运动写成「镜头切」 | 运动用「变为/运动成」，切用「切至」 |
| 13 | 源文本写「固定镜头」但视频实际有推进/运镜 | 运镜状态必须以视频实际为准，源文本错则改 |
| 14 | 视频有抽针/定格画面却写「慢速」 | 速度正常不写慢速，只在 shot 描述「有抽针画面展示」 |
| 15 | **跨镜头角色绑定错误（如正反打）** | 多镜头切换（尤其正反打镜头）后，各 shot 中 `<ID_x>` 的位置、朝向、引用须严格对应实际画面，不能错位 |
| 16 | **跨镜头描述衔接断裂** | 上下 shot 之间状态须逻辑衔接（结尾→开头连贯），同一动作不得错误拆分或重复描述 |
| 17 | **台词/字幕横跨镜头处理错误** | 🆕 台词横跨多镜头→每个 shot 只写实际听到的部分；字幕在多镜头同时存在→每个 shot 都要描述该字幕 |

> 游戏画面=非写实、时间戳单位等通用项见基础规则文档，此处不重复。

## 七、直播录频画面

> ⚠️ 该规则属于**基础规则**，已在《视频Caption规则文档--飞书实时更新版》「特殊说明及重要更新点」（0708 更新）中，本文档不重复。
> 要点速记：主播人像→前景（role+scene 重点描述）；屏幕共享→**只写背景**（「和风格」已删，不再写风格字段）；主要角色也要写 role；镜头写「直播录频固定镜头」。
- **游戏直播投屏 subtype**：真实人物以「小窗」出现 + 游戏大屏 → 真实人物写**主要角色（Role）**，游戏画面写**背景**。
