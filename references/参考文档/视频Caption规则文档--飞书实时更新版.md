# 数据类型

作业前先确认本题的数据类型，不同类型关注点不同：

- **t2v**（text to video，文本转视频）：无音频，不会涉及 `<sfx>`、`<speech>`、`<music>`、`<song>`，但 Role / Background / Style / Scene 等其他所有字段照常标注；
- **t2va**（text to video and audio，文本转视频和音频）：正常作业的 single 与 multi 均为此类，需要关注所有标注规则涉及的内容；
- **role指代专项**：只需核对 Role 描述、Role 数量、Scene 中 ID 指代、ID 动作语言描述是否准确。

# 二、筛选原则

- \<header\>：无需关注



- \<Event\>：务必要与\<Role\>中指代匹配，不匹配可筛去



- \<Role\> 

    - 角色数量匹配：视频中出现的主要角色数量和\<ID\_X\>不匹配可筛去；

    - 角色指代错误：角色特征指代错误可筛去；

    - 不用过多关注方位，如左右手、画面左右侧，大模型经常识别错误，需要作业员注意。



- \<Background\>

    - 背景数量不匹配可筛去；



- \<Music\>：无需关注



- \<Style\>：无需关注



- \<Scene\>

    1. 视频shot数量不匹配

    2. 听不懂的方言、粤语（0717更新）和外语（除英语），可筛去

# 四、具体标注规则

## 标注描述大原则

- **正确性：**Caption需要如实描述视频画面内容、动作、物体、人物、场景、方位等，例如“左边第二位女士的头上带着一顶蓝色的渔夫帽”，需要保证“左边”“第二”“女士”“蓝色”“渔夫帽”等词语都正确。

- **完整性：**需要涵盖视频的关键元素，包括主要的人物及其动作、重要物体、核心事件、场景环境、文字字幕等，需要站在“导演”的角度，将其需要展示的内容准确完整的描述出来。

- **简洁性：**在保证正确性、完整性的前提下，尽量避免冗余。

- **客观性：**基于视频内容客观描述，只描述画面出现的，不要脑补，避免主观臆断、情感倾向或评价

    - **除音效、音乐的描述**，其余标签**禁止描述**营造xxx氛围，烘托xxx气氛，表达xxx情感诸如此类的隐性信息描述

    - **不要否定词描述**：无背景音、无主要角色、无台词等任何没有的信息不要说明，不描述即可

    - 不要出现可能、推测、约、似乎、好像等词语，不要出现二选一的描述比如“或”、“/”

- **语言一致性：**机器识别的什么语言就保持什么语言描述，视频内容中出现外文、繁体字等信息要如实描述，不要翻译和转换

- **格式正确：**所有标签描述需要有头有尾,涉及的标签`<Scene>、<speech>、<section>、<shot>、<sfx>、<song>`举例，\<speech\>好的，再见\</speech\>`<Scene></Scene>`。所有标签务必前后包裹闭合！



## 各标签内容标注规范及注意事项

| 标签 | 字段含义 |
|------|---------|
| `<header>` | 描述视频基本信息 |
| `<Event>` | 描述画面主要事件，对画面内容做总结概括 |
| `<Role>` | 描述画面中的主要角色，包含数量、位置、年龄/性别/衣着打扮/配饰/语种音调等信息 |
| `<Background>` | 描述**画面**的背景信息，如场景布局、背景物体、光照等，以及画面存在的一些静态的水印等 |
| `<Music>` | 背景音乐风格及变化 |
| `<style>` | 描述**画面**的风格 |
| `<Scene>` | 描述随视频内容发展，**画面的**镜头运镜变化及**人**人物景交互的时序变化 |
| `<Shot>` | 镜头 |
| `<speech>` | 描述人物台词 |
| `<sfx>` | 短暂音效名词描述 |
| `<song>` | 歌词、曲风 |

### 各字段详细规则

- \<header\>

    - 若视频为**正常播放**速度：该字段直接**留空**。

    - 若存在明显**加速、慢速、延时**或其他特殊播放速度：**必须遵守格式**写：FPS\-\[帧率\]\. 这是一段\[**播放速度**\]播放的视频。   注：帧率选用大模型识别的速率

    - 若视频有部分属于特殊播放速度，如实描述即可，例：前半段正常播放，后半段为加速播放。

\</header\>



- \<Event\>

    - 视频**出处\+类型**。**仅用****一句话概括**描述该视频中最主要的可见事件。

\</Event\>

***注意点：***

1. **客观性原则：**如果出现营造XX氛围的多余的描述，直接删除即可

2. **避免ID指代：****不要用\<ID\_A\>\\\<ID\_B\>指代描述，可以说视频中有一男一女/有个男/女生、一名运动员向另一名运动员、一群人等等诸如此类的代词描述**

3. **视频出处：**若**非经典且作业员没听说过的**影视剧、综艺、游戏等（经典如甄嬛传、战狼、王者荣耀等），**无需注明片名，但是视频类型一定要保留**



- \<Role\> 

    - **若存在主要角色：****开头必须且只能按照格式写：视频中有\[数字\]个主要角色**

    - **描述主要角色\<ID\_x\>的外表特征：**包括年龄、性别、外貌、神态、穿着、配饰、音色、音调、口音

    - **若不存在主要角色：**此处标签内部直接留空，不做任何解释。

\</Role\>

***注意点：***

1. 描述为视频中有1个可清晰辨识且处于叙事核心的主要角色改成**视频中有1个主要角色****（直接删除冗余描述）**

2. 无需关注顺序要求，从 \<ID\_A\> 开始编号。每个对象格式为：\<ID\_A\>（简短特征）：……。每个对象写成一个连续段落，只描述可见特征，**禁止描述动作、情感、语言****。**

3. **赛事、演出等远景镜头：**画面中人多但是**没有非常明确清晰的主角**，Role**空白**即可。

能根据队服上的号码做区分也可以



- \<Background\>

    - **若存在背景：****开头必须且只能按照格式写：视频有\[数字\]个背景。**

    - **描述背景中存在的物体：**

        - 简洁交代**主要场景类型**：室内、室外、办公室、实验室等

        - **关键背景物及其物理空间布局**，并同步记录**光线来源**、**整体色彩色调****、**明暗效果

        - 同步记录伴随该场景实际听到的**背景环境声音和音效（****所有的声音都要用\<sfx\>\</sfx\>包裹****）：**音效举例：\<sfx\>嘈杂人声\</sfx\>、\<sfx\>雨声\</sfx\>、\<sfx\>雷电轰鸣声\</sfx\>、\<sfx\>观众席呐喊声\</sfx\>

        - 若场景中存在可见的**水印**，交代其**文字内容与画面位置**。

        - **弹幕**：**静态弹幕（较少出现）描述在Background，动态弹幕（常见）描述在Scene**

    - 若视频包含多个背景，请依背景编号分别进行描述。

\</Background\>

***注意点：***

1. 以上未提及信息，**如高清晰度必须删去**

2. 描述的音效应该是**视频拍摄现场**发出来的声音。Music音乐属于**后期**，**不属于拍摄现场的声音**，因此在music描述。**禁止**在此处描述music相关内容

** ****特殊案例：**若视频**拍摄现场**的手机、音响、乐器等设备传来的**声音、音乐****属于****background范畴****，****禁止****写在music**

3. **禁止记录字幕，字幕在Scene中描述**

4. 若水印在**变化**，如时间、数字滚动等，需要描述出来，比如：水印时间随视频播放从16:55变成16:58

5. **禁止****描述对话类型，如对白、独白等**

6. 若为**纯色背景**直接描述为：视频有1个背景，为纯色背景





- \<Music\>

    - 描述背景音乐伴奏的**风格、流派、核心乐器**，写明起初由特定乐器或节奏营造何种氛围，随后随何种画面事件转变为新乐器或新旋律**并突出情绪、氛围**

    - 若听辨不清或无音乐，此处标签内部请直接保持**留空**

\</Music\>

***注意点：***

1. 音乐描述专业要求较高，大模型文本描述无明显错误尽量不修改，明显错误需要修改，如**吉他声描述成钢琴声****、****紧张焦虑的氛围描述为轻松愉悦的氛围**

2. Music音乐描述内容是**后期加上的音乐**，用来渲染气氛，**禁止**描述拍摄现场发出的声音



- \<Style\>

    - **开头必须要先描述：**若为非写实，则写非写实风格，接下来再区分下级细分风格（注意删除"摄影摄像风格"），无需写写实风格。（0717更新）

    - **若为非写实风格：**例如卡通动漫、3D动画、数字插画、水墨、油画、复古胶片、赛博朋克、田园、奇幻、科幻、像素风、黏土动画、纸艺、游戏CG等，**必须要写上具体下级细分风格**。

    - **色调：**冷暖色调很明显的**必须**保留，不明显难判断的删除即可

    - **饱和度：**无严格要求，明显高/低饱和必须保留，不确定模糊的删除即可

    - **清晰度：无需且禁止描述，出现就要删去。**

\</Style\>



- \<Scene\> 

    **此处描述的内容总结一句话：用什么镜头、拍谁/什么东西、做什么、什么动作/神态/语气、发出什么声音、说了什么话，字幕写了啥**

    - **开头必须写：****视频包含\[数字\]个镜头。**

    - **镜头描述：*****必须用*****“景别 \+ 机位/视角 \+ 运镜状态 \+ 镜头”**的**规范格式**写出镜头类型，务必参考标准描述格式：举例：**中近景特写画面固定镜头、全景平视向前推进镜头**。

    \</Scene\> 

    ***注意点：***

    1. **描述整体原则：**描述时需注意将**主角运动和镜头运动**分开写，声音和动作穿插描述，让叙述更通顺。

    2. **\<Role\>引用：**引用对象时请直接使用 Role 中定义的 \<ID\_x\> ，**禁止重复描述** Role 中已经写过的**外观特征**；

    3. **声音描述：**

        1. **主角台词、对话内容、独白、旁白、画外音****：**使用 **\<speech\>****具体台词内容****\</speech\> **标签包裹实际听到的所有人声。必须严格按**音频**原始语言逐字听写台词，**禁止**受字幕影响

        2. 若为画外音，必须明确写明"画外音/拍摄者说道"，**严禁误归属****给画面内未开口的角色**。

若为视频中电视/手机等电子产品的画面传来的声音，需要说来源，例：电视中传来\<speech\>\</speech\>（0717更新）

        3. 在描述中如实体现**主角明显情绪、语速韵律（如急促/断断续续）及音量高低等核心人声信息。**

        4. 带有歌词且有人声演唱的歌曲：使用** \<song\>****歌词内容****\</song\> **标签包裹。

        5. 主角动作发出来的声音：使用** \<sfx\>****具体拟音与音效****名词****描述****\</sfx\>**标签包裹，且紧跟在触发它的动作或主体之后描述。**举例：**\<sfx\>衣物摩擦声\</sfx\>、\<sfx\>大笑声\</sfx\>、\<sfx\>叹息声\</sfx\>、\<sfx\>杯子与桌子碰撞声\</sfx\>）。**一定是听的清楚明确的声音才描述**

    4. **主角说话时嘴部动作禁止遗漏：**描述speech的时候，根据实际情况需要描述下**说话人的嘴巴张合**，**举例：**ID\_A张大嘴巴/嘴巴张合，兴奋地拉高音量急促地说\<speech\>xxxx\</speech\>

    5. 音频描述部分 ‘口型与声音严丝合缝、音画同步语速缓慢 ’ 这种**口型、语速相关的正确的描述可以保留**

    6. 地下室，室内回响的描述需要保留，比如“说话时伴有室内回响声”

    7. **画面可见文字与字幕记录规范：**

        1. 字幕应该要拆分出来单独描述，不能笼统的把所有字幕描述在一起（0715强调）

        2. **所有****字幕与台词一致，只需****最后总结****说明"字幕与台词内容一致"**

        3. 部分字幕与台词一致，需要在内容一致的台词描述后加上**"字幕与台词内容一致"，**需要在内容不一致的台词描述后加上**字幕内容**

        **字幕与台词不一致的描述格式为：**\<speech\>\.\.\.\.\.\.\</speech\>，字幕描述为"xxx"

        **字幕与台词一致的描述格式为：**\<speech\>\.\.\.\.\.\.\</speech\>，字幕与台词一致

        4. 若有**双语字幕**，如果中文一致，则需要描述英文字幕为"xxx"。如果中文不一致，则需要把中英字幕内容都描述出

        5. 如果画面中**没有出现文字或字幕**，则**直接忽略。**

    8. **弹幕**：**静态弹幕（较少出现）描述在Background，动态弹幕（常见）描述在Scene**

    9. **视频中人物台词听不清的，就保留大模型的生成内容，但是要注明不修改的原因，比如听不清、环境嘈杂等。**（0717删除）

## 特殊说明及重要更新点

- **游戏类视频描述**，游戏内设计大量专业、角色、技能、装备等名词，此类描述标准较高，**大模型识别的可以直接使用**。但要做好基础的颜色、材质、方位、错别字等细节的确认工作。



- **游戏、赛事、演出等画面中出现多个人物的视频描述：**role里出现的明显角色需要描述，比如boss、游戏使用角色、漏出球衣号码牌的球员、领唱等。NPC以及没有非常明确的清晰人物主体可以略过。



- **（0708更新）直播录频/投屏画面**：如果有主播人像画面则为前景，要在role和scene里重点描述；屏幕共享则为背景；如果有主要角色（如嘉宾、被采访者等）也要写在role中。镜头写直播录频/投屏固定镜头即可。



- **Background和Scene中音效的区分（0609晚21:44更新）：**

1. bg和scene中提到音效类描述**都要用\<sfx\>包裹**，bg和scene中**可以有重复音效**，但**不能遗漏**；（0717删除）

2. bg中更侧重于描述**视频背景音**（背景里发出的声音），比如嘈杂人声、画外音效果、环境底噪等；

3. scene中更侧重于描述**前景主体发出的声音**，主体做了什么动作发出什么声音，比如人走路发出的脚步声；

4. 对于一些主体发出的**笑声、叹气声**，如果有具体的字比如“哈哈哈”、“嘿嘿嘿”、“哎呀”这种，**也用sfx包裹**，比如发出开心的\<sfx\>哈哈声\</sfx\>





- **role描述方法：**

  - 同一人有妆造变化，如IDB、IDC为同一角色，在IDC的描述中直接表明，IDB和IDC是同一个人，然后描述妆造
  - role统一格式，\<ID\_\*\>（简单描述）

- **scene中音效音频类描述补充说明（0610更新）：**

1. \<speech\>必须以实际音频为准，且不能重复例如，要避免重复引用台词去串联描述。

2. 避免用第几秒去串联描述。
