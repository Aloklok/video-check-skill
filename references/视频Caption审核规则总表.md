# 视频Caption审核规则总表（单镜头 + 多镜头）

> **唯一审核规则总表。** 整合自：基础规则文档（飞书实时更新版）、多镜头审核规则、专业术语及样例、常见问题汇总。
> 审核时**只需读取本文件**即可，无需再读其他分散文档。
> 规则版本：综合 V1.0 | 整理日期：2026-07-10

---

# 0. 审核前置信息

## 0.1 数据类型（作业前先确认）

| 类型 | 含义 | 审核关注点 |
|------|------|-----------|
| **t2v** | text to video（文本转视频） | **无音频**，不涉及 `<sfx>`/`<speech>`/`<music>`/`<song>`，但 Role/Background/Style/Scene 等其余所有字段照常标注 |
| **t2va** | text to video and audio（文本转视频和音频） | 正常作业的 single 与 multi 均为此类，需关注**所有**标注规则 |
| **role指代专项** | 特殊审核模式 | **只核对**：Role 描述、Role 数量、Scene 中 ID 指代、ID 动作语言描述是否准确；其余字段不重点核查 |

## 0.2 平台操作

- 作业环节：标注 — 一审 — 二审
- 审核时**一审、二审每人只能有一个权限**（是一审就不能是二审，反之亦然）
- 首先选择**正常作业**，找到待审核任务开始作业
- 标注环节筛选项：筛选英文题、正常作业题目、废弃题目（**不要选挂起**）

## 0.3 标注描述大原则（适用于所有字段）

1. **正确性**：如实描述画面内容、动作、物体、人物、场景、方位等，每个修饰词都要正确
2. **完整性**：涵盖关键元素（主要人物及动作、重要物体、核心事件、场景环境、文字字幕），站在"导演"角度完整描述
3. **简洁性**：保证正确、完整前提下避免冗余
4. **客观性**：基于视频客观描述，只描述画面出现的，**不脑补、不主观臆断、不带情感倾向或评价**
   - **除音效、音乐的描述外，其余标签禁止描述**"营造xxx氛围、烘托xxx气氛、表达xxx情感"等隐性信息
   - **禁止否定词描述**：无背景音、无主要角色、无台词等没有的信息不要说明，不描述即可
   - 不出现"可能、推测、约、似乎、好像"等词，不出现二选一描述如"或"、"/"
5. **语言一致性**：机器识别的什么语言就保持什么语言，外文、繁体字如实描述，**不翻译不转换**
6. **格式正确**：所有标签描述有头有尾，涉及标签 `<Scene>`/`<speech>`/`<section>`/`<shot>`/`<sfx>`/`<song>` 务必前后包裹闭合

## 0.4 筛选 / 筛去原则

- `<header>`：无需关注
- `<Event>`：务必要与 `<Role>` 中指代匹配，不匹配可筛去
- `<Role>`：
  - 角色数量与 `<ID_X>` 不匹配 → 可筛去
  - 角色特征指代错误 → 可筛去
  - 不用过多关注方位（如左右手、画面左右侧，大模型常识别错误）
- `<Background>`：背景数量不匹配 → 可筛去
- `<Music>`：无需关注
- `<Style>`：无需关注
- `<Scene>`：
  1. `<Role>` 引用与视频不匹配 → 可筛去
  2. `<speech>`/`<sfx>`/`<song>`/`<music>` 中间内容无需过多关注
  3. 视频shot数量不匹配 → 可筛去
  4. 听不懂的方言、粤语和外语（除英语），可筛去
  3. **分点描述 → 直接筛去**

## 0.5 全局废弃条件

| 条件 | 说明 | 适用范围 |
|------|------|---------|
| **PPG** | 静态图片 + Ken Burns 效果（上下/左右平移 + 缩放）→ 废弃，不标注 | 多镜头特有，单镜头亦适用 |
| **第一人称视角游戏画面** | 第一人称 FPS 视角、驾驶视角等游戏录屏 → 直接废弃，无需审核各字段 | 单镜头、多镜头均适用 |
| **视频语言听不懂（含方言/小语种）→ 直接废弃** | 视频为听不懂的语言，含**粤语、闽南语、客家话等听不懂的方言**及**小语种/少数民族语言**，且无法据此校对字幕 → 直接废弃，不标注。歌词 `<song>` 等 speech 内容同样适用；语速过快听不清导致无法校正也适用。 | 单镜头、多镜头均适用 |
| **直播录频/投屏** | 见 §7.8（非废弃，是特殊标注模式） | — |

## 0.6 无音频场景处理（重要）

当视频只有 BGM、角色无台词时：
- **Scene 中不使用 `<speech>` 标签**（没有音频就没有台词）
- **删除所有音画同步相关描述**：如"音画同步""口型与台词吻合""配合语音节奏"等
- 嘴部动作改为描述"配合文字出现的节奏自然地开合"
- 画面字幕/对话框文字直接描述内容，**不套用**字幕与台词一致/不一致格式（无台词可参照）
- 若字幕分段依次出现，按出现顺序穿插在动作描述中
- **BGM 照常写在 Music**（不因其是游戏配乐而混淆为 Background 音效；游戏录屏没有"拍摄现场"概念）

# 1. Header 字段

## 1.1 基础规则
- 若视频为**正常播放速度**：该字段直接**留空**
- 若存在明显**加速、慢速、延时**或其他特殊播放速度：**必须遵守格式**写：
  `FPS-[帧率]. 这是一段[播放速度]播放的视频。` （注：帧率选用大模型识别的速率）
- 若视频有**部分**属于特殊播放速度，如实描述即可，例：前半段正常播放，后半段为加速播放

## 1.2 多镜头差异
- 变速片段用**"开头/中间/结尾部分"**定位，**不能用"镜头1、镜头2"**（Header 在镜头描述之前，此时尚无镜头概念）
- 对应 shot（Scene）中也须标明该片段是快速/慢速/延时
- 视频含**抽针/定格（stop-motion）画面但速度正常** → **不写"慢速"**，只在对应 shot 描述"有抽针画面展示"，Header 不标变速

# 2. Event 字段

## 2.1 基础规则
- 视频**出处 + 类型**，**仅用一句话概括**描述该视频中最主要的可见事件

## 2.2 注意点
1. **只写可见画面事件（铁律）**：Event 仅概括"看到的画面"里**最主要的可见事件**。**不写风格**（写实/非写实、画风等归 `<Style>` 字段）、**不写音频**（台词/音乐/音效归 `<speech>`/`<Music>`/`<sfx>`/`<song>`）。只保留 §2.1 要求的"视频出处+类型"一句（如"这是一部动漫视频""游戏录屏类视频"）——那是类型归类，不是风格/音频描述，须保留。
2. **客观性原则**：出现"营造XX氛围"等隐性/情感描述 → 直接删除
3. **避免 ID 指代**：不用 `<ID_A>`/`<ID_B>` 指代描述，用代词（"视频中有一男一女/有个男/女生、一名运动员向另一名运动员、一群人"等）
4. **视频出处**：若**非经典且作业员没听说过的**影视剧、综艺、游戏等（经典如甄嬛传、战狼、王者荣耀等），**无需注明片名，但视频类型一定要保留**。**⚠️ 平台（如 B站/抖音/YouTube）不算"视频出处"**——"出处"指内容类型归属（动漫/电影/综艺/游戏等，例子均为片名/游戏名），平台归属（logo/水印）归 `<Background>` 字段（总表 Background 平台 logo 规则），**不写进 Event**。
5. **电视剧/电影片段不出处通用类型**：若视频为电视剧/电影片段且无法识别具体剧名/片名——**不写**"该视频来自电视剧片段""这是一部电影片段"等通用来源描述，直接只写可见事件。若能识别具体剧名/片名则照常写。游戏直播、vlog 等其余类型不受此条影响，正常写视频出处+类型。

# 3. Role 字段

## 3.1 基础规则
- **若存在主要角色**：开头按格式写：**视频中有[N]个主要角色**（或 **视频中有[N]个核心对象**，二者均可，不作扣分）
- **描述主要角色 `<ID_x>` 的外表特征**：年龄、性别、外貌、神态、穿着、配饰、音色、音调、口音
- **若不存在主要角色**：标签内部直接留空，不做任何解释

## 3.2 注意点
1. "视频中有1个可清晰辨识且处于叙事核心的主要角色" → 改为 **"视频中有1个主要角色"**（直接删除冗余描述）
2. **无需关注顺序要求**，从 `<ID_A>` 开始编号。每个对象格式：`<ID_A>`（简短特征）：……。每个对象写成一个连续段落，只描述可见特征，**禁止描述动作、情感、语言**
3. **赛事、演出等远景镜头**：画面人多但无非常明确清晰的主角 → Role **空白**即可。能根据队服上的号码做区分也可以写 Role。
4. **动物 Role（新规则）**：画面中有多只动物作为主体时，每只分别用独立 `<ID>`（`<ID_A>`、`<ID_B>`…），括号内特征描述必须能区分开、**不能相同**。不可以用一个 ID 指代一群动物。
5. **同一人换装**：同一人有妆造变化（如 IDB、IDC 为同一角色），在后续 ID 的描述中直接表明"IDB 和 IDC 是同一个人"，然后描述妆造差异。
6. **Role 统一格式**：`<ID_*>`（简短特征）：……（括号内写简短特征，冒号后展开描述）
7. **口音/方言（有说话的角色必写）**：以下四种出现即必须描述——**东北话、粤语、四川话、上海话/吴语**。若角色有明显地方口音但无法确定具体类型，用「有地方口音」话术描述。全程未开口的角色可跳过此项。

## 3.3 多镜头差异
- **数量一致**：Header 写"视频中有 X 个主要角色"（或"核心对象"） → 下面 ID 须是 X 个。改动 ID 数量须同步改 Header
- **ID 顺序（多镜头特有）**：须按 A、B、C **连续**排列，**不能跳字母**（ABD 不行）
  > ⚠️ 冲突说明：单镜头场景写明"无需关注顺序要求"；多镜头按培训须 ABC 连续。两者适用场景不同，**多镜头以本要求为准**
- **核心对象可以是物品**：对讲机、工具等聚焦物品也可为主要角色；只露一只手/手指无其他特征 → 不算
- **同一人换装**：同一人换衣服 → 用**同一个 ID**，不新增；在 Role 描述注明服装变化时间点（"在 N~M 秒穿 XX，在 P~Q 秒穿 YY"），对应 shot 中也注明

# 4. Background 字段

## 4.1 基础规则
- **若存在背景**：开头**必须且只能**按格式写：**视频有[N]个背景。**
- **描述背景中存在的物体**：
  - 简洁交代**主要场景类型**：室内、室外、办公室、实验室等
  - **关键背景物及其物理空间布局**，并同步记录**光线来源**、**整体色彩色调**、**明暗效果**
  - 同步记录伴随该场景实际听到的**背景环境声音和音效（所有声音都要用 `<sfx></sfx>` 包裹）**：如 `<sfx>嘈杂人声</sfx>`、`<sfx>雨声</sfx>`、`<sfx>雷电轰鸣声</sfx>`、`<sfx>观众席呐喊声</sfx>`
  - 若场景中存在可见的**水印**，交代其**文字内容与画面位置**
  - 若画面存在**平台 logo 但认不出具体平台**：直接写**"平台名 + 颜色"**即可，例：`优酷蓝色 logo`，不必强行辨认
  - **弹幕**：**静态弹幕（较少出现）描述在 Background**；**动态弹幕（常见）描述在 Scene**
  - **背景中清楚可见的文字必须写全**（品牌名、标语、标识文字等，不得缩写或省略）
- 若视频包含多个背景，请依背景编号分别进行描述

## 4.2 注意点（含 Music/Background 分类，重点）
1. 以上未提及信息，**如"高清晰度"必须删去**
2. 描述的音效应该是**视频拍摄现场**发出来的声音。**Music 音乐属于后期，不属于拍摄现场的声音**，因此在 **Music 描述。禁止在此处描述 music 相关内容**
   - **⚠️ 特殊案例**：若视频**拍摄现场**的手机、音响、乐器等设备传来的**声音、音乐，属于 Background 范畴，禁止写在 Music**
3. **禁止记录字幕**，字幕在 Scene 中描述
4. 若水印在**变化**（如时间、数字滚动等），需要描述出来，例：水印时间随视频播放从 16:55 变成 16:58
5. **禁止描述对话类型**（如对白、独白等）
6. 若为**纯色背景**直接描述为：视频有1个背景，为纯色背景

## 4.3 音效分类规则（0609 晚更新 + 0610 更新，跨 Background/Scene/Music）

> 这是最易错的分类，务必牢记：**Background=现场环境音，Scene=主体动作音，Music=后期配乐**

| 类别 | 归属字段 | 内容 | 包裹标签 |
|------|---------|------|---------|
| 视频拍摄现场的环境底噪（嘈杂人声、画外音效果、环境底噪等） | **Background** | 现场背景音 | `<sfx>` |
| 前景主体做动作发出的声音（脚步声、衣物摩擦等） | **Scene** | 主体动作音 | `<sfx>` |
| 主体发出的笑声/叹气（有具体字如"哈哈哈""嘿嘿嘿""哎呀"） | **Scene** | 用 `<sfx>` 包裹，如 `<sfx>哈哈声</sfx>` | `<sfx>` |
| 后期配乐（BGM、伴奏） | **Music** | 后期音乐，渲染气氛 | 写在 `<Music>`，不包 `<sfx>` |
| 拍摄现场设备声/现场演奏乐 | **Background**（非 Music） | 现场声 | `<sfx>` |

> **⚠️ 边界（Music vs Scene 的 `<music>` 位置）**：
> - **后期配乐（BGM，含"全程伴随"类）只在顶层 `<Music>` 描述**，Scene 里**禁止复述/冗余写** BGM（否则既重复顶层 `<Music>`，又把"全片 BGM"错误钉在单个 shot）。
> - Scene 内 inline `<music>` / `<song>` **仅用于绑定该 shot 画面上事件的音乐**：如角色打开音响放歌、某情节点冒出歌词（`<song>`）、画面可见的现场演奏等**有画面来源的 diegetic 音乐**。
> - 判断口诀：**全片贯穿的后期 BGM → `<Music>`；某镜头里"画面上有声源"的音乐 → 该 shot 内 inline**。

**0609 规则**：
1. bg 和 scene 中提到音效类描述**都要用 `<sfx>` 包裹**，bg 和 scene 中**可以有重复音效，但不能遗漏**
2. bg 侧重**背景音**（嘈杂人声、画外音效果、环境底噪等）
3. scene 侧重**前景主体发出的声音**（主体做了什么动作发出什么声音）
4. 笑声叹气有具体字（哈哈哈、嘿嘿嘿、哎呀）也用 sfx 包裹

**0610 补充**：
1. `<speech>` 必须以实际音频为准，避免重复引用台词去串联描述
2. 避免用第几秒去串联描述（如"当 xx 说到……"）
3. `<sfx>` 要结合画面判断合理存在、明确可辨识的。**很细微的声音不描述**，清楚的一定要描述、要准确

**0716 补充（归属 + 核实）**：
1. **持续场景音**：持续的、贯穿全片的场景音（如持续不断的雨声、风声等）写在 Background 的 `<sfx>` 中即可，**不用在 Scene 中重复描述**。
2. **细微音效核实（铁律）**：Scene 的 `<sfx>` 中出现"轻微/细微/微弱/衣物摩擦声/衣服摩擦声"等词汇，**必须跟视频核实**，视频中实际听不到或不存在的一律删除。

## 4.4 多镜头差异
- **每个 shot 必须带背景**：多镜头下每个 shot 配一个背景编号（`<BG_1>`、`<BG_2>`…），AI 漏了须补，**绝对不能删**
- **跨镜头背景判断**：
  - 明显差异（方位大变、拍到不同区域）→ 不同背景，分别描述
  - 无明显变化或背景缺乏有效信息 → 视为同一背景，Scene 中不重复描述，但 Background 字段提一嘴
- **水印/Logo（多镜头特有处理）**：

  | 水印类型 | 处理方式 |
  |---------|---------|
  | 从头固定不变 | 仅首个镜头背景描述一次（位置+内容），后续不提 |
  | 中途出现/消失/变化 | 在对应 shot 描述（"哪边变在哪边提，没变不提"） |
  | 动态贴片文字（随台词/事件） | 在对应 shot 描述出现/消失，**不写 Background** |

  **动态贴片文字标注规则**（07011 补充）：
  - **位置**：在对应 shot 的 Scene 描述中写明，不写 Background（非静态水印）。
  - **内容要素**：须如实描述贴片的**位置、底色、文字内容**；若伴随音效，用 `<sfx>` 紧跟描述；贴片**消失**也需交代（如"随后贴片文字消失"）。
  - **格式模板**：`[位置]出现[底色]字的"[文字内容]"的贴片，同时伴随<sfx>"[音效内容]"的音效</sfx>，随后贴片文字消失……`

# 5. Music 字段

## 5.1 基础规则
- 描述背景音乐伴奏的**风格、流派、核心乐器**，写明起初由特定乐器或节奏营造何种氛围，随后随何种画面事件转变为新乐器或新旋律**并突出情绪、氛围**
- 若听辨不清或无音乐，此处标签内部请直接**留空**

## 5.2 注意点
1. 音乐描述专业要求较高，大模型文本描述无明显错误尽量不修改，明显错误需要修改（如**吉他声描述成钢琴声**、**紧张焦虑的氛围描述为轻松愉悦的氛围**）
2. Music 描述内容是**后期加上的音乐**，用来渲染气氛，**禁止描述拍摄现场发出的声音**（现场声归 Background）

# 6. Style 字段

## 6.1 基础规则
- **「写实风格」「非写实风格」标签均不写**：Style 字段**不再出现"写实风格"或"非写实风格"字眼**，直接写后续细分风格（若有）+ 明显色调/饱和度。写实/非写实判定只用作审核内部参考。
- **细分风格**（卡通动漫、3D动画、2D动画、数字插画、水墨、油画、复古胶片、赛博朋克、田园、奇幻、科幻、像素风、黏土动画、纸艺、游戏CG等）：画面有明显画风的**必须写**。
- **色调**：画面中有**明显**冷暖色调区分的（冷色调：偏蓝色/绿色；暖色调：偏红色/黄色），或**明亮度/对比明显**（如"色调明亮对比明显"），需要描述出来。不明显的无需描述。
- **饱和度**：画面中有**明显**饱和度区分的（高饱和/低饱和等），需要描述出来。不明显的无需描述。
- **清晰度**：无需且禁止描述，出现就要删去。
- **⚠️ 遗漏惩罚**：画面存在明显色调（如冷色调/暖色调）或明显细分风格（如2D动画），但标注未描述 → 当天全部数据打回返修。
- **⚠️ 边界（什么不写在此字段）**：Style 只描述**整体画面渲染风格**（细分画风 + 明显色调/饱和度）。**角色级细节不在此字段**：汗滴、表情/神态变化、具体动作、角色情绪等属 Role（神态/外表）或 Scene（动作/神态），不得写入 Style。唯一可入 Style 的"夸张"是描述**整体画风**为夸张风格（如卡通风格"角色造型夸张"），而非具体角色的细节。

## 6.2 多镜头差异
- 写实/非写实判定、游戏=非写实等基础规则见 §6.1
- **风格不一致的视频（多镜头特有）**：不写"写实风格""非写实风格"标签，直接写各段落的细分风格+明显色调/饱和度。若只有部分段落为非写实 → 描述为非写实段落对应的细分风格，用"前/中/后半段"定位（**不能用"镜头1、镜头2"定位**，Style 在镜头描述之前）。

# 7. Scene 字段

> 此处描述总结：**用什么镜头、拍谁/什么东西、做什么、什么动作/神态/语气、发出什么声音、说了什么话**

## 7.1 基础规则
- 开头**必须写**：**视频包含[N]个镜头。**
- **镜头描述格式**：必须用 **"景别 + 机位/视角 + 运镜状态 + 镜头"** 的规范格式，例：中近景特写画面固定镜头、全景平视向前推进镜头

## 7.2 注意点
1. **整体原则**：描述时需注意将**主角运动和镜头运动**分开写，声音和动作穿插描述，让叙述更通顺。**动作描述必须写全，不要漏掉关键动作。**
2. **`<Role>` 引用**：引用对象时直接使用 Role 中定义的 `<ID_x>`，**禁止重复描述** Role 已写过的**外观特征**
3. **声音描述**：
   1. **主角台词、对话、独白、旁白、画外音**：用 `<speech>具体台词内容</speech>` 包裹实际听到的所有人声。必须严格按**音频**原始语言逐字听写台词，**禁止受字幕影响**
   2. 若为**画外音**，必须明确写明"画外音/拍摄者说道"，**严禁误归属给画面内未开口的角色**。若为视频中电视/手机等电子产品的画面传来的声音，须注明来源，例：`电视中传来<speech>…</speech>`
   3. 在描述中如实体现**主角明显情绪、语速韵律（急促/断断续续）及音量高低**等核心人声信息
   4. 带有歌词且有人声演唱的歌曲：用 `<song>歌词内容</song>` 包裹
   5. 主角动作发出的声音：用 `<sfx>具体拟音与音效名词</sfx>` 包裹，且**紧跟在触发它的动作或主体之后**描述。例：`<sfx>衣物摩擦声</sfx>`、`<sfx>大笑声</sfx>`、`<sfx>叹息声</sfx>`、`<sfx>杯子与桌子碰撞声</sfx>`。**一定是听得清楚明确的声音才描述**
      6. **`<speech>` 归属镜头与画面文字对应（审核铁律）**：
         - **区分镜头**：多镜头下每个 `<speech>` 必须明确写在它真实所属的 `<Shot N>` 块内，且确为该 shot 实际听到的台词（对照 §7.5.9 跨镜头台词拆分）；不得将 A 镜头的台词错放到 B 镜头。单镜头仅一个镜头，天然满足。
         - **与画面文字对应**：`<speech>` 听录内容须与视频中实际出现的文字（字幕/可见文字）对应一致，按本 §7.2.7 逐句核对；speech 与字幕不符须显式标注，不得笼统带过（听录本身仍须以**音频**为准，§7.2.3.1，不被字幕带偏）。
   4. **speech 周围动作与描述规范（2026-07-16 扩展）**：
      - 张嘴/口型是独立面部动作，须在 speech 标签前单独描述（如「张开嘴巴」），不得省略或嵌入引导词中（「张嘴说道」不规范 → 应「张开嘴巴，说道<speech>…</speech>」）。
      - speech 引导只用简洁动词：「说道/喊道/问/回答/低声说/大叫」等，**禁止写长串动作修饰代替引导词**（如「张嘴用略显沙哑的普通话回应道」→ 拆成「张开嘴巴，用略显沙哑的普通话说道」）。口型描述须以画面实际为准。
      - **嘴巴张合/口型动作禁止遗漏**：说话人嘴部有任何可见动作（张大/微张/张合/嘴角上扬等）必须描述。
      - **镜头内角色任何可见的微小动作/表情不遗漏**：不仅嘴部，转头、眨眼、肢体微调、笑容变化等任何可见动作均须在 Scene 中描述，不得遗漏。
5. 音频描述部分"口型与声音严丝合缝、音画同步语速缓慢"这种**口型、语速相关的正确描述可以保留**
6. **地下室、室内回响**的描述需要保留，如"说话时伴有室内回响声"
   7. **画面可见文字与字幕记录规范**（字幕整体规则：写法 + 情况分类合并于此）：
      0. **⚠️ 字幕逐句绑定 speech 原则（必遵，2026-07-16 修订）**：
         - 字幕必须**逐句紧跟对应 `<speech>`**，写在每句 speech 之后，**禁止在镜头/shot 末尾统一罗列**（如笼统写成"字幕为A，B，C"）。
         - 多条字幕（A、B、C）→ 各自紧跟对应 speech：`speech A，字幕为A；speech B，字幕为B；speech C，字幕为C`，严禁整体合并罗列。
         - 字幕与台词一致 → 该句后写「，字幕与台词内容一致」；字幕与台词不一致 → 该句后写「，字幕描述为"xxx"」。**废止旧版"全部一致→结尾仅一句总结"写法。**
         - **字幕/画面文字按出现时序排列**：同一 shot 内多条字幕/画面文字，按其**出现的时间顺序**依次描述。若 speech 之前已出现独立字幕（非当前 speech 对应），须先在 speech 前描述该字幕内容及其外观，再按正常格式写 speech + 其对应字幕。
   1. 所有字幕与台词一致 → 每句 `<speech>……</speech>` 后均写「，字幕与台词内容一致」（逐句写，**不合并为一句结尾总结**）
   2. 部分字幕与台词一致 → 一致句后加「，字幕与台词内容一致」；不一致句后按格式加「，字幕描述为"xxx"」
      - 字幕与台词不一致格式：`<speech>……</speech>，字幕描述为"xxx"`
      - 字幕与台词一致格式：`<speech>……</speech>，字幕与台词内容一致`
   3. 若有**双语字幕**：中文一致则需描述英文字幕为"xxx"；中文不一致则需把中英字幕内容都描述出
   4. 画面中**没有出现文字或字幕** → 直接忽略
   5. **字幕语言/字体规则**：`<speech>` 中台词统一使用**简体中文**；字幕文本引用（如「字幕描述为'xxx'」中的 xxx）**保持画面原始语言**（繁体即繁体，不转换为简体）。
   **整体字幕情况分类**（与上方写法对应）：
   - **无字幕**
   - **有字幕，且字幕内容与音频完全一致**（语言一致，内容一致）→ 按 7.2.7.1 逐句写「，字幕与台词内容一致」（**不合并为结尾总结句**）
   - **有字幕但字幕内容与音频不完全一致**：
     - **字幕未被完整音频覆盖**（字幕内容多于音频可听到内容），例：字幕"测试一下模型效果"，音频"模型效果"
     - **字幕被完整音频覆盖，但额外包含翻译后的其他语言**，例：字幕"测试一下模型效果/Test the model's effect"，音频"测试一下模型效果"
     - **字幕是其他语言类型，翻译后可以和音频对齐**，例：字幕"测试一下模型效果"，音频"Test the model's effect"
8. **弹幕**：**静态弹幕（较少出现）描述在 Background**；**动态弹幕（常见）描述在 Scene**
9. **视频中人物台词听不清的**，保留大模型生成内容，但要注明不修改原因（如听不清、环境嘈杂等）
   - **示例**（处理部分清晰、部分模糊的台词）：以温和且略带关切的语速说道：`<speech>怀孕了妞儿们就走走 巴巴让自己放松啊，再放松啊好不好？要要特别让自己放松啊，放松放松，这样对你好对宝宝也好</speech>`。 **语速较快，无法辨别说话内容**。随后眼睛直视前方。

10. **方向/位置必须注明参照系（2026-07-16 新增）**：所有涉及方向或位置的描述（左/右/前/后/上/下），须注明参照系——「画面左/右侧」（镜头取景框内的方位）或「角色的左/右侧」（角色自身身体朝向下的方位）。示例：`ID_B位于画面左侧核心位置`（画面参照）vs `ID_B转向自己身体左侧`（角色参照）。仅写「转向左侧」而无参照 → 违规，须补全。

> **⚠️ 边界（Scene 里的「神态/情绪表情」vs「氛围/情感推断」）**：
> - **可写**：画面**直接可见**的表情/神态/情绪反应（微笑、瞪眼、震惊、大笑、皱眉、冷汗、语气等），属 §7.2 要求内容，缺失算问题（见 §9 缺失项）。
> - **禁写**：① 整体**氛围/气氛**（如"营造紧张氛围""烘托温馨气氛"）；② **纯主观评价/情感倾向**（如"显得很可怜""令人感动"）；③ **无可见对应的纯内心状态归纳**（如"她内心充满恐惧"而无表情支撑）。
> - 判断口诀：**看得见的表情/神态 → 写；看不见的氛围/内心推断 → 不写**。若一句既含可见描述又带推断（如"显得不知所措"），保留可见部分、删/改推断部分。

## 7.4 专业术语（景别 / 拍摄角度 / 运镜 / 拍摄手法 / 易混淆概念）

### 7.4.1 景别分类

| 景别 | 定义 | 表达氛围 |
|------|------|---------|
| 特写 | 肩部以上 | 拍摄局部细节，突出主题 |
| 近景 | 胸部以上 | 强调表情或动作 |
| 中景 | 膝盖以上 | 拍摄主体大部分，兼顾环境 |
| 全景 | 人物从头到脚完整入镜 | 环境作为背景补充 |
| 远景 | 人物全身可见，但环境仍占较大比例 | 强调大场景，人物较小，突出空间感 |
| 空镜 | 没有任何人物出现，纯景色和场景拍摄 | 强调拍摄场景和渲染氛围 |

### 7.4.2 拍摄角度分类

| 拍摄角度 | 定义 |
|---------|------|
| 仰拍 | 镜头从下向上拍摄 |
| 俯拍 | 镜头从高处向下拍摄 |
| 正面拍摄 | 镜头位于主体正前方 |
| 侧面拍摄 | 镜头位于主体侧面 |
| 背面拍摄 | 镜头位于主体背后，只能看到背影 |
| 过肩镜头 | 从一个角色肩膀后方拍摄对面角色，是对话场景最常用的机位。**不限于肩膀**：从角色头/脖后、或透过某人拍其后方人群等前后景构图，均算过肩镜头 |

### 7.4.3 运镜分类

| 运镜 | 定义 | 提示 |
|------|------|------|
| 固定镜头 | 摄像机的位置、拍摄角度都保持不变 | |
| 晃动镜头 | 相机在拍摄过程中位置和角度固定，画面有轻微晃动感 | 多为纪录片 |
| 手持静止镜头 | 画面固定且有轻微抖动，通过手持拍摄或后期模拟抖动，制造不稳定感 | 跟固定镜头区别在于：手持不稳定性使画面有微小抖动和移动 |
| 推镜头 | 摄影机靠近主体，主体周围环境减少 | |
| 拉镜头 | 与推镜头相反，摄影机远离主体，主体周围环境增多 | |
| 移镜头 | 摄影机在**水平方向**上移动。右移：从左到右；左移：从右到左 | 注意跟升降的区别 |
| 升镜头 | 摄影机沿**垂直方向向上**移动 | |
| 降镜头 | 与升镜头相反，摄影机沿**垂直方向向下**移动 | |
| 环绕镜头 | 摄影机围绕主体或某个固定点旋转 | 围绕主体环绕拍摄，展示主体与环境全方位关系，一般有明确主体 |
| 旋转镜头 | 摄像机沿镜头轴线旋转的拍摄方式，画面产生倾斜或旋转效果 | |
| 摇镜头 | 摄影机为圆心，镜头左右或上下旋转，展示更大范围的场景。上摇：从下往上；下摇：从上往下；右摇：水平向右；左摇：水平向左 | 与环绕镜头区别：环绕以被拍摄主体为圆心 |
| 跟镜头 | 摄影机跟随主体运动 | 一般有明确跟随主体（如跟人/动物前进后退），镜头有明显位移 |

### 7.4.4 常见拍摄手法

| 拍摄手法 | 定义 |
|---------|------|
| 希区柯克变焦 | 推镜头+变焦拉（或拉镜头+变焦推）同时进行，主体大小不变但背景急剧变化，制造眩晕/不安感 |
| 延时摄影 | 以较低频率间隔拍摄（如每5秒拍1张），后期合成视频，将漫长过程压缩为短时间（如日出、花开） |
| 手持摄影 | 摄影师手持摄像机拍摄，画面带自然晃动，增强真实感、临场感、纪实风格 |
| 慢动作 | 用高帧率（如120fps/240fps）拍摄，正常速度播放时动作变慢，展现细节、渲染情绪（如水滴、爆炸） |
| 快动作（加速） | 用低帧率拍摄或后期加速，压缩时间，表现忙碌、荒诞、喜剧效果 |
| 速度渐变（Speed Ramp） | 同一镜头中快慢速度交替变化，如从慢动作突然加速，节奏感极强，常用于 MV 和短视频 |
| 抽针/定格（Stop-Motion） | 画面以逐帧/定格方式呈现，人物或物体呈"跳变"式运动，似被抽去中间帧；常与延时结合。速度正常时**不写"慢速"**，只在对应 shot 描述"有抽针画面展示" |

### 7.4.5 易混淆概念区分

| 易混淆项 | 正确标注依据 |
|---------|------------|
| 镜头拉近 vs 推镜 | 镜头拉近 = 仅**变焦**（相机不动）；推镜 = 物理移动相机 |
| 晃动 vs 抖动 | 晃动 = 无规律运镜；抖动 = 手持相机/车载相机无意识移动 |

## 7.5 多镜头差异（Multi-Shot，重点）

### 7.5.0 核心概念：镜头切 vs 镜头运动（判定根基）
- **镜头切（硬切）= 多镜头**：画面瞬间跳变到另一角度/场景，通常来自不同机位，后期剪辑切换。满足其一即为镜头切：①非一镜到底；②画面忽然变换、无过渡动画、瞬间从 A 跳到 B
- **镜头运动 ≠ 多镜头**：同一台摄像机连续物理移动/变焦（推拉摇移跟），画面连续变化无跳跃
- **口诀**：连续变化 = 运动 = 单镜头；突然跳变 = 切 = 多镜头。**"镜头切"与"镜头运动"绝对禁止混用**

### 7.5.1 Shot 格式与结构
```
视频包含[N]个镜头。

<shot_1 start="0.0s" end="4.8s">
<section>景别 + 机位/视角 + 运镜状态 + 镜头
[描述：什么人/东西、做什么、说什么、发出什么声音]
</section>
</shot_1>

<shot_2 start="4.8s" end="9.5s">
<section>镜头切至 + 景别 + 机位/视角 + 运镜状态 + 镜头
[描述...]
</section>
</shot_2>
```
- 每个 shot 标签格式：`<shot_N start="x.xs" end="x.xs">`，内容用 `<section>...</section>` 包裹，结尾 `</shot_N>`
- **时间格式铁律**：`start`/`end` 的时间值必须是**一位小数 + 小写 `s`**，例如 `0.0s`、`4.8s`、`12.3s`。**禁止**写成 `Xs`/`Ys` 占位、`0.0S`/`4.8S`（大写 S 错）、`5s`（无小数）或 `5.00s`（两位小数）。即固定 `x.xs` 形态。
- 开头结尾 shot 编号须一致（shot_3 结尾须 `</shot_3>`）
- `<section>` 为官方格式标签，**不要删除或改写法**

### 7.5.2 镜头切字规则
- **第 1 个镜头**：不写切字，直接"XX镜头"
- **第 2 个及之后**：开头必须写切字（"镜头切至XX镜头""切为XX镜头"）
- **删除"通过硬切"四字**：原始文本常写"通过硬切切至XXX"，审核删"通过硬切"，只留"镜头切至XXX"
- 带**特效转场**（叠画/滑动/弹入等）时保留特效描述："通过叠画转场切至XXX镜头"

### 7.5.3 镜头内运镜描述（不新开 shot）
同一 shot 内有运镜（推/拉/摇/移/跟）→ **不新开 shot**，在当前 shot 内按时间顺序描述：
```
镜头切至近景摇镜头
[描述摇的过程中看到什么]
镜头快速推进变为正面全景
[描述推进后的画面]
```
**关键**：同 shot 内运动用"镜头运动成""变为""随着镜头…"，**绝对不能用"镜头切"**

### 7.5.4 时间戳校验
- 每个 shot 起止时间须与实际切点一致，误差 ≤ **0.5 秒**，超则改
- **结尾时间最易出错**（AI 文本常比实际短），务必核对
- 单位注意：`0:04` = 4 秒，不是 0.04 秒
- **分/秒格式歧义**：`1.6` = 「1分6秒」，`6.1` = 「6秒多（约6.1秒）」。秒数须与视频实际一致，不能视频 7 秒写成 8.5

### 7.5.5 跨镜头台词拆分
- 一句话跨两个 shot → 按**每个 shot 内实际听到的台词**拆分，不把整句塞进一个 shot

### 7.5.6 分屏画面

| 情况 | 镜头数 | 处理方式 |
|------|--------|---------|
| 景别/角度一致 | 1 个镜头 | 分左右/上下写，一个 shot 两个背景 |
| 景别/角度不一致 | 2 个镜头 | 分别描述左右/上下各是什么镜头 |

### 7.5.7 特殊案例
- **画面卡顿但内容不变**：角度、背景都没变，仅播放卡顿/跳帧感 → 视为 **1 个镜头**
- **镜头结尾被遮挡变全黑**：Shot 内要描述「画面逐渐被遮挡/变全黑」的变化过程，不能只写开头不看结尾
- **PPG**：见 §0.5 废弃条件

### 7.5.8 跨镜头角色绑定与衔接（07011 补充，多镜头高频易错）
- **跨镜头角色绑定（如正反打）**：多镜头切换（尤其正反打 over-the-shoulder reverse shots）后，各 shot 中 `<ID_x>` 的**位置、朝向、引用须严格对应实际画面**。例：shot A 中 `<ID_A>` 在左前景、`<ID_B>` 在右背景 → 切到 shot B 后应反过来；不得把同一角色在两 shot 中写错位置或错绑 ID。审核时逐一核实每个 shot 的角色绑定。
- **跨镜头描述衔接**：上下 shot 之间的状态须**逻辑衔接**——上一 shot 结尾状态 → 下一 shot 开头状态连贯，无跳跃、无矛盾、无重复；同一动作不得被错误拆分到两 shot 或重复描述。

### 7.5.9 跨镜头台词与字幕处理（07011 补充）
- **台词横跨多个镜头**：一句对话/台词的音频跨越了两个或以上镜头 → 每个 shot 中**只描述该镜头内实际听到的部分**，不得在未听到的 shot 中写完整句。
- **字幕横跨多个镜头**：一条字幕（subtitle/caption）在多个镜头的画面中同时存在 → **每个 shot 都需要描述该字幕**（因字幕是画面元素，每个可见 shot 均应体现）。

## 7.6 转场类型（多镜头特有）

| 转场类型 | 说明 |
|---------|------|
| 硬切 | 瞬间跳变无过渡，写"镜头切至XX镜头" |
| 叠画 | 两画面短暂重叠后切换 |
| 滑动 | 新画面从一侧滑入覆盖旧画面 |
| 弹入/淡入淡出 | 画面逐渐显现或消失 |
| 特效转场 | 特定视觉特效过渡 |
| 万花筒转场 | 万花筒效果切换 |
| 百叶扇转场 | 百叶窗效果切换 |
| 运动转场 | 甩镜、剧烈旋转、跳切等方式 |

> 带特效转场格式：`通过XX转场切至YY镜头`（与硬切的"只写镜头切至"区分）

## 7.7 多镜头易错点汇总

| 序号 | 易错点 | 正确做法 |
|------|--------|---------|
| 1 | 把镜头运动当成镜头切 | 连续变化=运动，突然跳变=切 |
| 2 | 漏写镜头切字 | 除首镜头外，每个 shot 开头必须写 |
| 3 | 保留"通过硬切"四字 | 删除，只留"镜头切至" |
| 4 | 漏删 shot 的背景 | 每个 shot 必须有对应背景 |
| 5 | ID 数量改动后忘同步 Header | 前后一致 |
| 6 | ID 跳字母（ABD） | 必须 ABC 连续 |
| 7 | 同一人换装开新 ID | 同人同 ID，描述中写服装变化 |
| 8 | 水印变/没变都反复写 | 不变仅首镜头一次，变了的在对应镜头写 |
| 9 | 场景音全堆最后一句 | 分配到对应各背景 |
| 10 | Style/Header 用"镜头1、镜头2"定位 | 用"前/中/后段"，这些字段在镜头之前 |
| 11 | 一句话跨镜头不分拆 | 按每个 shot 实际听到的拆分 |
| 12 | 同 shot 内运动写成"镜头切" | 运动用"变为/运动成"，切用"切至" |
| 13 | 源文本写"固定镜头"但视频实际有推进/运镜 | 运镜状态必须以视频实际为准，源文本错则改 |
| 14 | 视频有抽针/定格画面却写"慢速" | 速度正常不写慢速，只在 shot 描述"有抽针画面展示" |
| 15 | 跨镜头角色绑定错误（如正反打） | 多镜头切换后各 shot 的 `<ID_x>` 位置/朝向/引用须严格对应实际画面，不得错位 |
| 16 | 跨镜头描述衔接断裂 | 上下 shot 状态须逻辑衔接，同一动作不得错误拆分或重复描述 |
| 17 | 台词/字幕横跨镜头处理错误 | 🆕 台词横跨多镜头→每个 shot 只写实际听到的部分；字幕在多镜头同时存在→每个 shot 都要描述该字幕 |

## 7.8 直播录频/投屏画面（0708 更新，0716/0717 补充，单镜头基础规则）

- 主播人像画面为**前景**，要在 **Role 和 Scene 里重点描述**
- **屏幕共享则为背景**
- 如果有**主要角色**（如嘉宾、被采访者等）也要写在 **Role** 中
- 镜头写**直播录频/投屏固定镜头**即可
- **游戏直播投屏 subtype**：真实人物以"小窗"出现 + 游戏大屏 → 真实人物写**主要角色（Role）**，游戏画面写**背景**
- **真实人物 + 游戏录屏同时存在（0716 补充）**：Role 写真人 + 游戏角色（如果有明显角色）；镜头写「直播录频固定镜头」或真人对应的镜头即可。

| 11 | 需要对片段内容动态描述，不能只描述静态画面 | 例：一个人走在街上突然蹲下系鞋带，关键帧为系鞋带，不能只针对系鞋带画面描述，要描述整个走动→蹲下→系鞋带过程 |
| 12 | 客观事实陈述错误 | 例：主体之间的方位关系描述有误 |
| 13 | AI 生成的结果看起来未做任何更改的 | 正确描述：（需按实际补充） |

## 9.3 运镜问题

| 序号 | 出现的问题/规则要求 | 错误解析 |
|------|-------------------|---------|
| 1 | 定义不清晰——【跟镜头】 | 推镜头和跟镜头的区别：需理解拍摄意图，明显跟随主体运动就写跟镜头 |
| 2 | 定义不清晰——【降镜头】与【平移】混淆 | 注意不是下移，平移通常指水平方向左右移动 |
| 3 | 定义不清晰——【摇镜头】 | "摇"不是指晃动的意思，而是以摄像机为圆心镜头左右或上下旋转，展示更大范围的场景 |
| 4 | 明显不是【固定镜头】但没有做修正 | |
| 5 | 描述不能是各类运镜的罗列，需要体现镜头变化 | 可用："先 xx 运镜，再 xx 运镜" / "xx 运镜结合 xx 运镜" / "xx 运镜切至 xx 运镜"，无固定格式要求 |

# 10. 字段含义速查

| 标签 | 字段含义 |
|------|---------|
| `<header>` | 描述视频基本信息 |
| `<Event>` | 描述画面主要事件，对画面内容做总结概括 |
| `<Role>` | 描述画面中的主要角色，包含数量、位置、年龄/性别/衣着打扮/配饰/语种音调等信息 |
| `<Background>` | 描述**画面**的背景信息，如场景布局、背景物体、光照等，以及画面存在的一些静态的水印等 |
| `<Music>` | 背景音乐风格及变化 |
| `<style>` | 描述**画面**的风格 |
| `<Scene>` | 描述随视频内容发展，**画面的**镜头运镜变化及**人**人物景交互的时序变化 |
| `<Shot>` | 镜头 |
| `<speech>` | 描述人物台词 |
| `<sfx>` | 短暂音效名词描述 |
| `<song>` | 歌词、曲风 |
